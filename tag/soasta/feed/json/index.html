Soasta on how they deal with real user monitoring in AngularJS:

\n

\nFirst, we now listen for AngularJS routing events, such as $routeChangeStart. Once we see that the visitor is about to view new content, we start listening in preparation for gathering performance metrics for the soft navigation.

\n

Second, and most importantly, when the page is first loading, and when a soft navigation starts, we begin monitoring the HTML document for any new downloadable resources that get inserted. We do this via a MutationObserver.\n

\n

Real user monitoring techniques for single page apps are becoming a big deal, and don’t easily have a one size fits all solution.
