Ever wonder what would happen if you dipped SQLite into Redis? You get Thredis, the peanut butter cup of databases:

\n

\nThredis embeds an in-memory SQLite database within Redis to enable a
\nsophisticated level of SQL (joins, sub-selects, etc, all supported),
\nas well as introduces multi-threaded processing to take advantage of
\nSMP systems.\n

\n

This is one wild hack, check out one of the examples:

\n

[sourcecode]
\nredis 127.0.0.1:6379> zadd foo_zset 1 a 2 b 3 c
\n(integer) 3
\nredis 127.0.0.1:6379> sql "create virtual table zset using redis (foo_zset)"
\nOK
\nredis 127.0.0.1:6379> sql "select * from zset"
\n1) 1) "key"
\n 2) "val"
\n2) 1) "1.0"
\n 2) "a"
\n3) 1) "2.0"
\n 2) "b"
\n4) 1) "3.0"
\n 2) "c"
\n[/sourcecode]

\n

More examples are listed at http://thredis.org/, source code is available at https://github.com/grisha/thredis.
