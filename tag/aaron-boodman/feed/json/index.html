Aaron Boodman on How Chromium Works with “No Branches”:

\n

\nOn many projects, it\u2019s common to branch the source code to work on major new features. The idea is that temporary destabilization from the new code won\u2019t affect other developers or users. Once the feature is complete, its branch is merged back into trunk, and there is usually a period of instability while integration issues are ironed out.

\n

This wouldn\u2019t work in Chrome because we release every day. We can\u2019t tolerate huge chunks of new code suddenly showing up in trunk because it would have a high chance of taking down the canary or dev channels for an extended period. Also, the trunk of Chrome moves so fast that it isn\u2019t practical for developers to be isolated on a branch for very long. By the time they merged, trunk would look so different that integration would be difficult and error-prone.\n

\n

I’m a long time fan of this approach, where branching is the exception and not the rule.

\n
