The ramp up to HTTP/2 seems to have also triggered a renewed interest in web server software. I recently came across Caddy, a web server written in Go. From the FAQ:

\n

\nFundamentally, Caddy aims to make websites easier by lowering the barrier to entry. Like other web servers, Caddy is for system administrators and web developers, but Caddy is also tailored for designers and writers who don’t want to deal with a lot of the technical cruft that traditionally comes along with setting up a server.

\n

Caddy has different goals and advantages. Nginx is first and foremost a capable reverse proxy, whereas Caddy’s primary goal is to be an easy-to-use static file web server. Caddy desires cross-platform independence and portability without the need for custom builds.\n
