pup, the three sentence description nails is:

\n

\npup is a command line tool for processing HTML. It reads from stdin, prints to stdout, and allows the user to filter parts of the page using CSS selectors.

\n

Inspired by jq, pup aims to be a fast and flexible way of exploring HTML from the terminal.\n

\n

If you have used jq, this is going to feel familiar, just for an HTML context. With the option to output the results as JSON, you could find your self using pup in conjunction with jq.

\n

There are several examples in the README, here are a few of my own.

\n

List all of the IMG tags from cnn.com:
\n[shell]
\ncurl -s https://www.cnn.com/ | pup ‘img’
\n[/shell]

\n

Now get it back as a JSON array:
\n[shell]
\ncurl -s https://www.cnn.com/ | pup ‘img json{}’
\n[/shell]

\n

List all of the LINK attributes that have a rel=\"preload\" attribute:

\n

[shell]
\ncurl -s https://www.cnn.com/ | pup ‘link[rel="preload"]’
\n[/shell]

\n

I could see pup and jq becoming standard utilities on Unix-like systems.

Last year I mentioned a nifty little command line tool called jq. It had a very specific purpose: make it easy to process JSON. That same line of thinking inspired pup, from Eric Chiang:

\n

\npup is a command line tool for processing HTML. It reads from stdin, prints to stdout, and allows the user to filter parts of the page using CSS selectors.\n

\n

Here is a little taste of how this works:

\n

[sourcecode]
\ncurl -s https://wordpress.com/ |
\npup link
\n[/sourcecode]

\n

That will return all of the link elements on the WordPress.com home page. You can easily tighten it down to just the shortlink:

\n

[sourcecode]
\ncurl -s https://wordpress.com |
\npup link[rel=shortlink]

\n

<link rel="shortlink" href="http://wp.me/1">
\n[/sourcecode]

\n

One final step further, get just the href value:

\n

[sourcecode]
\ncurl -s https://wordpress.com |
\npup link[rel=shortlink] attr{href}

\n

http://wp.me/
\n[/sourcecode]

\n

I like it.

\n

Pup is open source, written in go and available at https://github.com/EricChiang/pup.
